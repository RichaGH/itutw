# Ответ

Вот пошаговая инструкция по настройке NFS-сервера и клиента на AlmaLinux 9.7 и выше. Она подготовлена с учётом различий между версиями протокола, чтобы вы, как начинающий специалист, могли сразу выбрать подходящий вариант.

Главное, что вам нужно знать перед началом: **NFSv3 и NFSv4 версии — это два разных подхода к организации сетевого доступа к файлам**, и ваш выбор должен основываться на потребностях в безопасности, производительности и совместимости.

---

### 1. Введение: NFSv4 или NFSv3? В чём разница?

| Характеристика | NFS версия 3 (NFSv3) | NFS версия 4 (NFSv4 / v4.1 / v4.2) |
| :--- | :--- | :--- |
| **Состояние соединения** | **Без сохранения состояния (Stateless)**: сервер не отслеживает состояние клиентов и блокировки. | **С сохранением состояния (Stateful)**: сервер и клиент отслеживают состояние соединения и блокировки. |
| **Блокировки файлов** | Требует дополнительных протоколов (NLM, MOUNT) вне NFS. | **Встроены в протокол NFS**, не нужны сторонние службы. |
| **Безопасность** | Базовая, на основе UID/GID и IP-адресов. | Расширенная. Поддержка **Kerberos**, улучшенные **ACL** (списки контроля доступа). |
| **Порты брандмауэра**| Требуется открывать несколько динамических портов для `rpcbind`, `mountd`, `nlockmgr` и т.д. | Использует только один статический порт **2049**. Упрощает настройку firewall. |
| **Скорость и производительность** | **Выше** в операциях с большим количеством мелких файлов (метаданные), так как протокол более "лёгкий". | Может быть немного медленнее из-за больших накладных расходов на обеспечение безопасности и stateful-соединений. |
| **Кто использует** | Широко используется там, где нужна максимальная совместимость со старым оборудованием и высокая скорость. | Современный стандарт для новых проектов, особенно при высоких требованиях к безопасности и контролю доступа. |

### 2. Подготовка к установке

Пакет `nfs-utils` предоставляет все необходимые утилиты как для сервера, так и для клиента. Его установка обязательна на обеих машинах.

1.  **Обновление системы**: Всегда начинайте с обновления пакетов.
    ```bash
    sudo dnf update -y
    ```

2.  **Установка `nfs-utils`**:
    ```bash
    sudo dnf install nfs-utils -y
    ```

### 3. Настройка NFS-сервера

Эти шаги выполняются на машине, которая будет предоставлять свой диск в общий доступ.

**Шаг 1:** Создайте директорию, которой хотите поделиться, и задайте ей необходимые права. В качестве владельца часто используют `nobody:nobody`.
```bash
sudo mkdir -p /srv/nfs_share
sudo chown -R nobody:nobody /srv/nfs_share
sudo chmod 755 /srv/nfs_share
```

**Шаг 2:** Настройка служб аутентификации и отображения ID.
*   Отредактируйте **/etc/idmapd.conf**: Этот файл критически важен для правильной работы NFSv4, так как он отвечает за отображение имён пользователей и групп между клиентом и сервером.
    ```bash
    sudo nano /etc/idmapd.conf
    ```
    Раскомментируйте или добавьте строку `Domain`. В качестве значения рекомендуется указать доменное имя или имя вашего сервера.
    ```systemd
    Domain = your-server-hostname
    ```

**Шаг 3:** Определите общие каталоги в файле `/etc/exports`.
```bash
sudo nano /etc/exports
```
Здесь вы указываете, какой каталог, для каких клиентов и с какими параметрами будет доступен. Добавьте строку в следующем формате:
```bash
/srv/nfs_share client_IP_or_subnet(rw,sync,no_subtree_check)
```
*   **`/srv/nfs_share`**: Путь к общему каталогу.
*   **`client_IP_or_subnet`**: IP-адрес конкретного клиента (например, `192.168.1.100`) или подсеть (например, `192.168.1.0/24`). Использовать `*` для всех не рекомендуется в производственной среде.
*   **`rw`**: Права на чтение и запись. **`ro`** — только чтение.
*   **`sync`**: Сервер подтверждает операцию только после записи данных на диск. Это повышает надёжность, но может снизить производительность.
*   **`no_subtree_check`**: Отключает проверку поддеревьев, что решает некоторые проблемы с доступом к файлам.
*   **`no_root_squash`**: По умолчанию доступ пользователя `root` на клиенте приравнивается к пользователю `nobody` на сервере в целях безопасности. Эта опция снимает это ограничение. Используйте только при крайней необходимости!


**Шаг 4:** Запустите и включите службу NFS.
```bash
sudo systemctl enable --now nfs-server
```

**Шаг 5:** Примените конфигурацию экспорта.
```bash
sudo exportfs -rav
```
Флаг `-a` (all) экспортирует все каталоги из `/etc/exports`, `-r` (re-export) перечитывает конфигурацию, `-v` (verbose) выводит подробную информацию.

### 4. Настройка Firewall и SELinux

Эти два компонента часто становятся причиной проблем у новичков, так как они блокируют сетевой доступ по умолчанию.

*   **Firewalld**: В зависимости от выбранной версии NFS, вам нужно открыть разные порты.

    *   **Для NFSv3**: Требуется открыть несколько служб.
        ```bash
        sudo firewall-cmd --permanent --add-service={nfs3,rpc-bind,mountd}
        sudo firewall-cmd --reload
        ```
    *   **Для NFSv4**: Достаточно открыть всего один порт - 2049. Если ваша конфигурация использует только NFSv4, можно отключить поддержку NFSv3 в файле `/etc/nfs.conf` для усиления безопасности.
        ```bash
        sudo firewall-cmd --permanent --add-service=nfs4
        sudo firewall-cmd --reload
        ```

*   **SELinux**: Чтобы не отключать защиту, а просто разрешить NFS экспортировать каталоги, используйте переключатель `nfs_export_all_rw`.
    ```bash
    sudo setsebool -P nfs_export_all_rw on
    ```

### 5. Настройка NFS-клиента

Эти шаги выполняются на компьютере, который будет подключать удалённую папку.

1.  **Установите `nfs-utils` (если не сделали этого ранее)**.
2.  **Создайте точку монтирования**:
    ```bash
    sudo mkdir -p /mnt/nfs_client_share
    ```

3.  **Смонтируйте общий каталог вручную (для проверки)**. Замените `NFS_SERVER_IP` на IP-адрес вашего сервера.
    ```bash
    # Для NFSv3 или v4
    sudo mount -t nfs NFS_SERVER_IP:/srv/nfs_share /mnt/nfs_client_share

    # Для NFSv4 (более строгий синтаксис, но часто работает и первый вариант)
    sudo mount -t nfs4 NFS_SERVER_IP:/srv/nfs_share /mnt/nfs_client_share
    ```

4.  **Добавьте запись в `/etc/fstab` для автоматического монтирования при загрузке**.
    ```bash
    sudo nano /etc/fstab
    ```
    Добавьте строку:
    ```bash
    NFS_SERVER_IP:/srv/nfs_share /mnt/nfs_client_share nfs4 defaults,_netdev 0 0
    ```
    *   `_netdev`: Указывает системе, что это сетевое устройство, и монтировать его нужно только после поднятия сети. Это обязательная опция, иначе загрузка может зависнуть.

| Если вам нужно... | То ваш выбор — NFS... | Почему |

| **Максимальная совместимость** со старыми Unix-системами.| **v3** | NFSv3 — более старый и широко распространённый стандарт. |
| **Высокая производительность** в операциях с множеством мелких файлов.| **v3** | Stateful-протокол NFSv4 создаёт ощутимую дополнительную нагрузку на метаданные. |

| **Улучшенная безопасность** и поддержка ACL/Kerberos.| **v4** | Включает встроенные механизмы безопасности и более гибкие списки контроля доступа. |
| **Встроенная поддержка блокировок файлов** без дополнительных служб.| **v4** | Протокол управляет блокировками самостоятельно, не полагаясь на внешние NLM-сервисы. |
